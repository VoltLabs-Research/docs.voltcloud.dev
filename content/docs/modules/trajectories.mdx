## Overview

Trajectories are the core workflow in VOLT. A trajectory represents a molecular dynamics simulation that the platform parses, stores, and renders into an interactive 3D model. From a single upload, you can navigate frames, view per-atom data in the Particles tab, apply color coding, filter particles, run analysis plugins, and share results with your team.

## Trajectory Listing

The main view shows all trajectories for your team, organized by folders. Each trajectory displays its name, status, frame count, atom count, and creation date. You can create folders to group related simulations, move trajectories between folders, and use search to find specific files.

![Trajectory Listing](/assets/modules/trajectories/trajectory-listing.png)

## Uploading a Trajectory

To upload a trajectory, click the **Upload** button and select one or more simulation files. VOLT accepts any file — it is not restricted to specific extensions. After uploading, the trajectory goes through an automated processing pipeline:

| Status | Description |
|---|---|
| **Queued** | The trajectory is waiting for a worker to become available |
| **Processing** | Files are being parsed and metadata is being extracted |
| **Rendering** | 3D GLB models are being generated for each frame |
| **Completed** | The trajectory is ready for viewing and analysis |
| **Failed** | An error occurred during processing — check the error details for more information |

![Upload Trajectory](/assets/modules/trajectories/upload-trajectory.png)

## 3D Viewer

Once a trajectory reaches the **Completed** status, you can open it in the 3D viewer. The viewer renders atoms as spheres in a WebGL scene using GLB models streamed from your cluster's object storage.

- **Frame Navigation**: Use the timeline slider to scrub through timesteps, or press play to animate the simulation.
- **Orbit Controls**: Rotate, pan, and zoom the camera to inspect the structure from any angle.
- **Particles Tab**: Open the **Particles** tab in the timeline area to view a tabular listing of all atoms — including ID, type, position coordinates, and any per-atom properties present in the dump file.
- **Simulation Cell**: The bounding box of the simulation cell is displayed as a wireframe overlay, showing periodic boundary conditions.

![3D Viewer Controls](/assets/modules/trajectories/3d-viewer-controls.png)

## Particles

The **Particles** tab (located in the timeline area at the bottom of the viewer) displays a tabular listing of every atom in the current frame. Columns include atom ID, type (color-coded), X/Y/Z coordinates, and any additional per-atom properties extracted from the dump file (e.g., velocities, forces, potential energy, custom compute outputs). You can scroll through the full atom list with infinite loading.

## Color Coding

Color coding maps a per-atom property to a color gradient across the 3D model, letting you visually identify patterns like stress concentrations or temperature distributions.

1. Open a trajectory in the 3D viewer and click **Color Coding**.
2. Select the per-atom property you want to visualize (e.g., `c_pe`, `vx`, `type`).
3. Review the min/max statistics for the selected property.
4. Choose a color gradient (e.g., blue-to-red, viridis, plasma).
5. Click **Apply** — the viewer re-renders the model with the new color mapping and adds a scene artifact.

![Color Coding](/assets/modules/trajectories/color-coding.png)

## Particle Filters

Particle filters let you isolate subsets of atoms based on property conditions. You can preview the filter before applying it to confirm the selection.

| Action | Description |
|---|---|
| **Highlight** | Keep only the matching atoms visible, hiding everything else |
| **Delete** | Remove the matching atoms from the scene, keeping everything else |

Filters support standard comparison operators (`=`, `!=`, `>`, `<`, `>=`, `<=`) applied to any per-atom property. After applying a filter, a new scene artifact is created so you can toggle it on and off without losing the original view.

## Scene Artifacts

Every color coding, particle filter, or plugin exposure that generates a 3D model is stored as a **scene artifact**. The artifacts panel in the viewer lists all generated models for the current trajectory and timestep. You can toggle each artifact on and off, switch between them, or remove ones you no longer need.

Scene artifacts can originate from three sources: color coding operations, particle filter actions, and analysis plugin exposures that produce GLB models.

![Scene Artifacts](/assets/modules/trajectories/scene-artifacts.png)

## Sample Simulations

If you want to explore the platform without uploading your own data, VOLT provides sample simulations. Click **Sample Simulations** from the trajectory listing to browse pre-built LAMMPS trajectories. You can download them and then upload them to your workspace manually — this way you learn the upload workflow firsthand.

## Task: Upload and visualize a trajectory

1. From the trajectory listing, click **Upload** and select a simulation file.
2. Wait for the processing pipeline to complete — the status will move through **Queued**, **Processing**, **Rendering**, and finally **Completed**.
3. Click the trajectory name to open it in the 3D viewer.
4. Use the timeline slider to scrub through frames or press play to animate.
5. Open the **Particles** tab to view per-atom data.

**Expected outcome:** The trajectory renders as an interactive 3D model with frame navigation, orbit controls, and the Particles tab showing atom data.

## Task: Apply color coding to a property

1. Open a trajectory in the 3D viewer and click **Color Coding**.
2. Select the per-atom property you want to visualize (e.g., `c_pe`, `vx`, `type`).
3. Choose a color gradient (e.g., blue-to-red, viridis, plasma).
4. Click **Apply** to re-render the model with the new color mapping.
5. Open the **Scene Artifacts** panel and confirm the new color coding artifact appears in the list.

**Expected outcome:** Atoms in the 3D viewer are colored according to the selected property and gradient, and a toggleable scene artifact is created.

## Task: Filter atoms by property

1. Open a trajectory in the 3D viewer and open the **Particle Filters** panel.
2. Select a per-atom property and set a comparison condition (e.g., `type = 1` or `c_pe > 0.5`).
3. Preview the filter to confirm the selection matches the atoms you expect.
4. Choose **Highlight** to keep only matching atoms visible, or **Delete** to remove them from the scene.
5. Click **Apply** to execute the filter.

**Expected outcome:** The 3D viewer updates to show only the filtered subset (or removes it), and a new scene artifact is created that you can toggle on and off.

## Task: Share a trajectory publicly

1. From the trajectory listing, open the trajectory you want to share.
2. In the trajectory settings or details panel, toggle **Public Sharing** on.
3. Copy the generated shareable link.
4. Open the link in an incognito or logged-out browser window to verify access.

**Expected outcome:** Anyone with the link can view the trajectory in the 3D viewer without needing a VOLT account.

## Additional Features

- **Download**: Export the original trajectory files from the platform at any time.
- **Public Sharing**: Mark a trajectory as public to generate a shareable link that anyone can use to view it — no account required.
- **Folder Organization**: Create folders, rename them, and move trajectories between folders to keep your workspace organized.
- **Volt AI Integration**: Ask [Volt AI](/docs/modules/volt-ai) to list trajectories, get details, run analyses, or manage your simulation data using natural language.

<Callout type="info">
Trajectory data is stored on your team's cluster in MinIO object storage. The VOLT server streams 3D models on demand — it does not keep copies of your simulation files.
</Callout>
