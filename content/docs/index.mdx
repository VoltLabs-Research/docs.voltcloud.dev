## What's VOLT?

![Introducing VOLT](/assets/getting-started/introducing-volt.png)

VOLT is a web platform for **materials science research teams** running LAMMPS molecular dynamics simulations. If you spend your day writing LAMMPS input scripts, visualizing atomic trajectories, and analyzing simulation results, VOLT centralizes that workflow into a single collaborative environment.

### Who VOLT is for

- Graduate students and postdocs running MD simulations.
- Research groups that need to share trajectories, results, and publications.
- Teams that want one-click analysis without managing local tool installations.

### What you can do

- **Visualize trajectories** — Render LAMMPS dumps in an interactive 3D viewer supporting 100M+ atoms. Scrub frames, inspect atoms, apply color coding, and filter particles.
- **Run analyses** — Execute analysis plugins (Coordination Analysis, DXA, Grain Segmentation, etc.) on your trajectories with a few clicks. Results include tables, charts, and 3D models.
- **Write publications** — Use the built-in collaborative LaTeX editor with live compilation, figure generation, and AI assistance.
- **Script custom workflows** — Create Jupyter notebooks with a full scientific Python stack (`numpy`, `scipy`, `matplotlib`, `pandas`, `voltsdk`, `ovito`, `ase`, `pymatgen`, `MDAnalysis`, and more) for post-processing beyond built-in plugins.
- **Deploy containers** — Spin up Docker containers on your clusters to run LAMMPS simulations or host custom tools.
- **Import from HPC** — Pull trajectory files from remote servers over SSH without manual downloads.
- **Collaborate in real time** — Chat, whiteboards, shared trajectories, and team management with role-based access.
- **Use AI across the platform** — Connect any of 13 AI providers (OpenAI, Anthropic, Google, etc.) and use Volt AI's 71 tools to manage resources with natural language.

### What VOLT is not

VOLT is not a replacement for OVITO, VMD, or other standalone visualization tools. Those tools focus on desktop trajectory visualization and analysis. VOLT includes visualization and analysis as modules within a broader platform that also handles collaboration, publishing, scripting, and infrastructure management. Think of it as a "Microsoft Office for materials science" — just because it has a presentation tool doesn't make it Canva.

### Key concepts

| Concept | Meaning |
|---|---|
| **Team** | A workspace that owns trajectories, clusters, members, and settings. You can create multiple teams. |
| **Cluster** | A machine (your laptop, a server, etc.) that stores your data and runs analyses. Each team needs at least one. |
| **Trajectory** | A LAMMPS simulation file (dump or data) that VOLT parses, stores, and renders into an interactive 3D model. |
| **Plugin** | An analysis workflow made of connected nodes. Plugins run on your cluster and produce exposures. |
| **Exposure** | An output file produced by a plugin — tables, charts, or 3D models viewable in the platform. |

### How it works

VOLT itself does not store your simulation data. Everything lives on your team's clusters. The VOLT server acts as a real-time proxy — when you request data through the UI, it forwards the request to the appropriate cluster. This makes the platform horizontally scalable: need more resources, just connect another cluster.

## Creating your VoltID

Go to [app.voltcloud.dev/auth/sign-in](https://app.voltcloud.dev/auth/sign-in) to get started.
You can sign in using your **Google**, **GitHub**, or **Microsoft** account.
If you prefer not to use any of those options, you can enter your email address and continue. If an account already exists with that email, you'll be asked for your password — otherwise, you'll be prompted to fill in your details to create a new account.

![Sign in page](/assets/getting-started/sign-in.png)

## Onboarding

### Team Creation

After creating your account, the first step is to create a team. A team is your **workspace** — it owns your trajectories, clusters, analyses, and members.

![Team creation](/assets/getting-started/onboarding-team-creation.png)

You can rename your team later, and you can create additional teams at any time.

### Connecting a Cluster

A cluster can be your personal computer, a spare machine, or a dedicated server.
Each team must have at least one cluster.

![Connect a cluster](/assets/getting-started/onboarding-connect-cluster.png)

After selecting your cluster type, assign it a name:

![Cluster name](/assets/getting-started/onboarding-cluster-name.png)

Finally, run the provided command in your terminal. If you selected **server**, run the command via SSH on your server. If you're using your **personal computer**, open a terminal and run it there.

![Cluster command](/assets/getting-started/onboarding-cluster-command.png)

<Callout type="warn">
On Windows, make sure to open the terminal with **administrator privileges**. The command will install Docker — if the installation fails, try [installing Docker manually](https://docs.docker.com/get-started/get-docker/).
</Callout>

![Running the cluster command](/assets/getting-started/onboarding-cluster-command-run.png)

Once the command finishes, you're all set to start using VOLT.

![Cluster connected](/assets/getting-started/onboarding-cluster-connected.png)

## Optional: Downloading VOLT

You can use VOLT directly from your browser. There is also a desktop app with identical functionality — some people simply prefer a dedicated window.

| Platform | Download |
|---|---|

| **Windows** | [VoltClient-Latest.exe](https://repo.voltcloud.dev/releases/windows/VoltClient-Latest.exe) |
| **Linux (Debian-based)** | [VoltClient-Latest.deb](https://repo.voltcloud.dev/releases/linux/debian/VoltClient-Latest.deb) |
